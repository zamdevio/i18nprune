# Changelog (placeholder)

This folder is reserved for release changelogs starting with the first public release.

Web companion page: `apps/web` route `/changelog` (footer-linked).

## Planned usage

- Keep one entry per release (for example `v0.1.0.md`).
- Summarize user-facing changes first, then migration notes and breaking changes.
- Link each release note to related docs/ADR pages when relevant.
- Mirror key notes into the web changelog data model (`apps/web/src/routes/pages/changelog/data.ts`) when needed.

Until the first release, this file acts as the canonical placeholder.
