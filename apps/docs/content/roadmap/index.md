# Roadmap

This page is a **high-level** view of where the project is headed. Priorities shift with real usage; treat it as guidance, not a contract.

## Near term

- **Richer review** — Structured reports for missing keys, drift, and optional unused-key signals (without locking to one vendor).
- **Provider abstraction** — Formal translator interface beyond the current Google `gtx` path.
- **Docs & CI** — Published site, optional GitHub Action for test + build on each push.

## Medium term

- **Performance** — Large monorepos: smarter scanning boundaries, caching where safe.
- **Editor / DX** — Optional JSON Schema or LSP-adjacent hints (no commitment yet).

## How we track work

This **Roadmap** stays **public product** direction — not sprint-by-sprint checklists. Issues + milestones are tracked alongside code contributions when scopes crystallize.
