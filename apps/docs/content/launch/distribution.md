# Distribution & adoption (scheduling)

Companion to [README](.). Use this when you are **scheduling** posts, changelog copy, or community announcements — not for web implementation backlog (**`apps/web`** and maintainer roadmap; see [Roadmap](../roadmap)).

## Checklist (when you ship a release)

- [ ] **Changelog** — user-visible fixes and features; link to docs for details.
- [ ] **Release post** — short outline: problem, one command to try, link to docs + npm.
- [ ] **Optional channels** — Reddit, Dev.to, X — one thread per channel; avoid duplicate wall-of-text.

## Relation to launch README

The main [launch README](.) covers **positioning**, **landing/docs/npm alignment**, and **README rewrite** plans. This file is only **when and where** you broadcast — timing and channels, not product build tasks.
